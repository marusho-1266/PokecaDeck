## 技術的解析詳細

### データ取得方法

この解析は、マークダウンファイル「ポケカデッキコードからの情報取得.md」に記載されている技術的手法に基づいて実施されました。

#### 1. HTML構造の解析

公式サイトのデッキ表示ページ（`result.html`）には、以下の2つの主要なデータソースが存在します：

- **Hidden Input Fields**: フォーム内のhidden input要素に、カード情報が `cardId_count_mainFlag` 形式でエンコードされています
  - `deck_pke`: ポケモン
  - `deck_gds`: グッズ
  - `deck_tool`: ポケモンのどうぐ
  - `deck_sup`: サポート
  - `deck_sta`: スタジアム
  - `deck_ene`: エネルギー

- **JavaScript PCGDECK Object**: ページ内のJavaScriptに、カード名、画像URL、詳細情報が含まれています

#### 2. データ抽出プロセス

1. ブラウザでページにアクセス
2. DOMからhidden input要素を取得
3. エンコードされたデータをパース（`-`で分割、`_`で分割）
4. JavaScriptの`PCGDECK`オブジェクトからカード名と画像URLを取得
5. カード詳細ページのURLを構築（`/card-search/details.php/card/{cardId}/`）

#### 3. 取得されたデータ構造

各カード情報には以下の属性が含まれています：

- **cardId**: 5桁のカードID（例: 48624）
- **name**: カード名（短縮版）
- **fullName**: カード名（完全版、セット情報含む）
- **count**: デッキ内の枚数
- **category**: カードのカテゴリ
- **mainFlag**: メインフラグ（1: 通常、9: メイン）
- **imageUrl**: カード画像のURL
- **detailUrl**: カード詳細ページのURL
